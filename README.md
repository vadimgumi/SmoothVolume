# Smooth Volume — плавные шаги громкости без root

Идея: система квантует громкость до ~15 шагов. Это приложение перехватывает
физические кнопки громкости через AccessibilityService и добавляет несколько
программных под-шагов между каждым реальным системным шагом, используя
`LoudnessEnhancer`, подключённый к глобальному аудио-миксу (сессия 0).

Технику подключения эффекта к сессии 0 без root подсмотрел у RootlessJamesDSP:
приложению нужно разрешение `android.permission.CAPTURE_AUDIO_OUTPUT`,
которое НЕ выдаётся обычным способом (это signature-permission), но которое
можно выдать себе через Shizuku командой `pm grant` — потому что Shizuku
исполняет команды от имени shell UID, а shell исторически имеет право
выдавать это разрешение (используется для тестирования звука через adb).

## Сборка

1. Открой папку `SmoothVolume` как проект в Android Studio (Giraffe/Koala и новее).
2. Дай Gradle синхронизироваться — подтянет `rikka.shizuku:api`.
3. Собери и установи APK на телефон (`Build > Build APK(s)` или `./gradlew installDebug`).

## Настройка на телефоне

1. Убедись, что Shizuku запущен (у тебя уже настроен через Wireless ADB — просто
   активируй сервис в приложении Shizuku).
2. Открой Smooth Volume → нажми "Выдать разрешение через Shizuku".
   - Подтверди запрос разрешения Shizuku, если появится диалог.
   - Нажми кнопку ещё раз — выполнится `pm grant ... CAPTURE_AUDIO_OUTPUT`.
   - В статусе должно появиться "Разрешение выдано".
3. Нажми "Открыть настройки Спец. возможностей" → найди "Smooth Volume" →
   включи сервис.
4. Проверь: покрути громкость кнопками во время воспроизведения музыки —
   переходы между системными щелчками должны звучать более плавно, звук не
   должен резко скакать.

## Настройка чувствительности

В `SmoothVolumeEngine.kt`:
- `SUBSTEPS` — сколько промежуточных уровней добавляется на 1 системный шаг
  (сейчас 4; больше — плавнее, но кнопку придётся нажимать чаще для реального
  переключения громкости).
- `MAX_GAIN_MB` — насколько сильно эффект может поднять/опустить громкость
  внутри под-шага (в милибелах, сейчас ~6 дБ).

## Известные ограничения (честно)

- **Если производитель прошивки (в т.ч. Transsion/XOS) закрыл возможность
  shell выдавать `CAPTURE_AUDIO_OUTPUT`** — `pm grant` вернёт ошибку, и
  `LoudnessEnhancer` не подключится к глобальной сессии. В этом случае
  `SmoothVolumeEngine.start()` поймает `SecurityException`, `enhancer` останется
  `null`, и кнопки будут работать как обычно (сервис просто не вмешивается) —
  без падений, но и без плавности. Проверить получится только на реальном
  устройстве.
- **AccessibilityService должен быть жив в фоне** — на XOS настрой то же самое,
  что и для Termux:API (отключить оптимизацию батареи, DuraSpeed и т.д.),
  иначе систему может убить сервис, и кнопки вернутся к обычному поведению.
- Это добавляет программный gain, а не реальные новые дискретные уровни в
  системе — то есть индикатор громкости в шторке будет по-прежнему прыгать по
  15 старым значениям, плавность будет именно в звучании, не в UI-полоске.
- Не тестировалось мной на реальном устройстве — собери и проверь, вернусь к
  правкам если что-то поведёт себя не так на твоём Infinix/Redmi.
